# 📘 Intro 페이지 개발 요약 (박재현 담당)

본 문서는 팀 프로젝트에서 **재현(jaehyun)** 이 담당한 **인트로(Intro) 페이지 개발 영역**을 정리한 내용입니다.  
인트로 페이지는 프로젝트의 첫 화면으로, 전지적 독자 시점(전독시)의 분위기·세계관·핵심 테마를 사용자에게 직관적으로 전달하는 역할을 수행합니다.

---

## 🎨 1. 개발 목표

- 사용자 첫 진입 화면에서 **전독시의 감성·분위기 전달**
- 작품 소개 / 기본 정보 / 핵심 테마 / 스토리 흐름을  
  **일관된 UI 디자인 시스템으로 구조화**
- Git 협업 브랜치 전략에 따라 intro 영역을 **전담 개발 및 유지보수**

### ✅ ④ 스토리 흐름(Story Flow) 3단계 섹션 전면 리뉴얼

인트로 페이지의 핵심이 되는 3단계 스토리 흐름을 UX 중심으로 재설계:

#### **STEP 1 — 유일한 완독 독자, 김독자**
- 3,000회차 완독이라는 설정을 시각적으로 표현  
- 본문 강조 문장(Bold) 및 라인 간격 최적화  

#### **STEP 2 — 소설 속 멸망 시나리오가 현실화**
- 갑작스러운 시스템 메시지·도깨비 등장 흐름 설명  
- 스토리 변화 지점을 한눈에 읽히도록 구성  

#### **STEP 3 — 독자가 세계에 개입하기 시작**
- 김독자의 선택 변화 → 본격 개입 구조로 흐름 연결  
- 전체 UI를 카드 구조로 재정비

📌 적용한 시각적 강화 요소  
- STEP 번호를 **그라데이션 원형 배지**로 커스텀 제작  
- 텍스트 크기 16px / line-height 1.9로 가독성 최적화  
- 깊은 다크톤 박스 + 안쪽 그림자 적용  
- 각 STEP을 독립된 카드처럼 보이도록 레이아웃 정비  
- CSS 구조를 `intro.css`에서 별도 관리
